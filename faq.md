# Instructions for manually editing IFX 

Q: How do I remove fastener catalogs so only INCH shows up?
A: IFX decides what appears from the files under your configured `ifx/parts` tree (the same folder Creo points to for the library). Open `ifx_catalogs/ifx_catalogs.txt` in a text editor. That index lists catalog names under section headings such as `#screws`. Remove the lines for any catalog you do not want listed (for example, catalogs you use only for metric/MM content), save the file, and restart Creo or refresh IFX as needed. If you no longer need a catalog at all, you can also remove or rename its matching file in the same folder (for example `some_catalog.txt`) so it is not referenced, and delete or archive the corresponding `.dat`/`.prt` pairs in `ifx_fastener_data` only if you are sure nothing else references those fastener names. There is no single “INCH only” switch—visibility is controlled by which catalogs remain in the index and what each catalog file lists.

Q: How do I create an extra fastener in the INCH catalog?
A: Treat it as three coordinated edits: catalog listing, data file, and Creo part. (1) In `ifx_catalogs/<your_catalog>.txt`, add a new line under the correct section heading (`#screws`, `#washers`, `#nuts`, `#inserts`, or `#pins`) with the fastener family name in lowercase, no spaces, one name per line. Keep a newline after section headings so new lines do not run into the heading. (2) In `ifx_fastener_data`, you need a `<name>.dat` file (and matching `<SYMBOL>.prt` files) that follow IFX’s INSTANCE table layout. The practical way without the app is to start from an existing family: copy a similar `.dat` from `ifx_fastener_data` or copy the template pair from `ifx_fastener_templates` (see “templates” below), rename consistently, and edit the tab-separated rows so each row’s **SYMBOL** matches the Creo part filename (no version suffix in the name IFX uses). (3) Copy or save the Creo models as `<SYMBOL>.prt` into `ifx_fastener_data` with the same base name as each SYMBOL. PTC’s IFX documentation describes the `.dat` INSTANCE format and how templates map to parts.

Q: How do I edit the fasteners I create?
A: **`.dat` files:** Edit with a plain text editor (UTF-8). They hold INSTANCE rows and parameters IFX reads at placement time. Add or change rows carefully—columns are tab-separated; keep **SYMBOL** unique across the whole library and consistent with part filenames. **`.prt` files:** Open in Creo to change geometry, dimensions, references, or custom parameters; if you use `param_relations.txt` in `ifx_fastener_data`, align parameter names with what you drive from the `.dat` fields. **Catalog `.txt` files:** Only needed when renaming or removing a fastener from a catalog’s list; edit the catalog file under `ifx_catalogs`, not the index, unless you are adding or removing whole catalogs.

Q: How do I make new fasteners from templates?
A: **How to use templates (manual workflow):** Under `ifx_fastener_templates` (beside the app when using the tool, or under `parts/ifx_fastener_templates` inside an imported library), each template lives in a folder such as `screws/screw_01/` with `.dat`, `.prt`, preview `.gif` files, and sometimes unit-specific parts. To create a new family by hand, pick the closest template folder, copy it, rename files and internal references so **SCREWTYPE** (or the equivalent type field in your `.dat`) still points at a valid template type number IFX understands, and place the working `.dat`/`.prt` copies in `ifx_fastener_data` with names matching your catalog entry and SYMBOL values. Icons optional: `ifx_fastener_icons` mirrors the template type for UI thumbnails.
